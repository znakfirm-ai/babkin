# JSX/TSX Safety Guard (MUST)

Минимальный гайд, чтобы не ловить TS1005/TS1381 и артефакты вроде “) :(” в UI.

## Частые ошибки
- Неполный тернарник: `cond ? (<div />)` без `: ...`.
- Висячие `) : (` после копипаста.
- Незакрытые теги или несбалансированные `{}`, `()`.
- Дубли ключей в `style` (два `border`, два `padding`), признак битого копипаста.
- Ветка тернарника с несколькими корневыми элементами без обёртки.

## Чеклист перед пушем
- В изменённой зоне проверил баланс `()`, `{}`, JSX‑тегов.
- Тернарники: обе ветки есть, нет “висячих” хвостов.
- При сложных условиях убран вложенный тернарник — логика вынесена в переменную.
- `style` без дублирующихся ключей.
- Корневой элемент в каждой ветке тернарника.
- Обязательно: `cd frontend && npm run build` — билд зелёный.

## Рекомендации по структуре JSX
- Не переписывай большие блоки/модалки целиком, меняй только нужные строки.
- Избегай вложенных тернарников; лучше подготовить переменную `const content = ...` и вставить `{content}`.
- Для условий без `else` используй `{cond && (<...>)}` вместо тернарника.
- В больших кусках JSX держи один корневой элемент на ветку.

## Как чинить, если уже сломали
1) Локализуй участок (по строкам из TS1005/TS1381).
2) Восстанови один корневой элемент на ветку.
3) Закрой все теги и скобки; убери дубли `) : (` и двойные `}`.
4) Убедись, что тернарник завершён `) : (...) )`.
5) Запусти `npm run build` перед push.
