## Правила разработки

### Безопасность (всегда)
- SQL injection — защищать через параметры, не строки
- Валидация входных данных через Pydantic
- Никаких секретов в коде — только через переменные окружения

### Код
- Async/await везде где можно
- Обработка ошибок — понятные сообщения
- Middleware для логов и авторизации
- REST стандарт: GET (получить), POST (создать), PUT/PATCH (изменить), DELETE (удалить)

### Задачи 3+ шага
1. Записать план в `tasks/todo.md`
2. Сверяться с планом перед каждым шагом
3. После шага: ✅ → сверить → следующий шаг
4. Если что-то пошло не так — СТОП, переплановать

### Файлы и пути (критично)
- Если не видишь файл в системе или не уверен в пути — НЕ ПРИДУМЫВАЙ
- Сначала `ls` или `find` чтобы убедиться что файл существует
- Только потом предлагать правки или писать импорты

### Правило трёх попыток
- Если сделал 3 попытки исправить баг и тесты всё равно падают — СТОП
- Выдать хозяину отчёт: что пробовал, что упало, где предположительно затык
- Не продолжать бесконечно

## Создание FastAPI проекта с нуля

Когда задача "создай бекенд" или "инициализируй FastAPI":

```bash
mkdir backend && cd backend
python -m venv venv
source venv/bin/activate          # Mac/Linux
pip install fastapi "uvicorn[standard]" sqlalchemy alembic \
  pydantic python-dotenv psycopg2-binary redis httpx \
  python-jose "passlib[bcrypt]" python-multipart loguru
pip freeze > requirements.txt
alembic init alembic
```

**Структура которую создаёшь:**
```
backend/
  app/
    __init__.py
    main.py           ← FastAPI app, подключение роутов
    api/
      __init__.py
      routes/         ← по одному файлу на группу эндпоинтов
    models/           ← SQLAlchemy модели
    schemas/          ← Pydantic схемы (входные/выходные данные)
    services/         ← бизнес-логика
    core/
      config.py       ← настройки из .env
      database.py     ← подключение к PostgreSQL
      security.py     ← JWT, хеширование паролей
  alembic/            ← миграции
  tests/
  requirements.txt
  .env.example        ← шаблон переменных (без реальных значений)
  .env                ← реальные значения (в gitignore)
```

**Минимальный `app/main.py`:**
```python
from fastapi import FastAPI
from app.api.routes import router

app = FastAPI()
app.include_router(router)

@app.get("/health")
async def health():
    return {"status": "ok"}
```

## Примеры

**Задача:** "сделай апи для регистрации пользователя"
**Что делаешь:**
1. Записываю план: модель → роут → валидация → тест
2. DB-SUBAGENT: создать таблицу users с нужными полями
3. Создаю Pydantic модель UserCreate
4. Создаю POST /users/register с проверкой что email уникальный
5. Защита: хешировать пароль, не хранить открытым
6. Показываю curl запрос и ответ — доказательство что работает

**Задача:** "добавь кеш для списка товаров"
**Что делаешь:**
1. CACHE-SUBAGENT: настроить Redis ключ для товаров
2. В эндпоинте GET /products: сначала смотреть Redis, потом базу
3. При изменении товара — сбрасывать кеш
4. Показываю что второй запрос отвечает быстрее
