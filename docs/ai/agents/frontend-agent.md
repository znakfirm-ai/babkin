## Правила разработки

### Веб (React)
- Компоненты — маленькие и переиспользуемые
- Состояние — управлять аккуратно, не плодить лишнее
- Формы — валидация на фронте перед отправкой
- WebSocket — подключать только когда нужны живые данные

### Мобилка (React Native + Expo)
- Тестировать через ExpoGo на реальном устройстве
- Push уведомления — настраивать через Expo SDK
- Сборка (ExpoBuild) — только когда приложение готово к публикации

### Задачи 3+ шага
1. Записать план в `tasks/todo.md`
2. Сверяться с планом перед каждым шагом
3. После шага: ✅ → сверить → следующий шаг
4. Если что-то пошло не так — СТОП, переплановать

### Файлы и пути (критично)
- Если не видишь файл в системе или не уверен в пути — НЕ ПРИДУМЫВАЙ
- Сначала `ls` или `find` чтобы убедиться что файл существует
- Только потом предлагать правки

### Правило трёх попыток
- Если сделал 3 попытки исправить баг и тесты всё равно падают — СТОП
- Выдать хозяину отчёт: что пробовал, что упало, где предположительно затык
- Не продолжать бесконечно

## Самопроверка (ОБЯЗАТЕЛЬНО после написания кода)

Перед тем как сказать "готово" — применить `/code-review-excellence` к своему коду:

**Чеклист React:**
- [ ] Нет бесконечных useEffect (проверить зависимости)
- [ ] Таймеры и подписки очищаются в return () => cleanup
- [ ] Формы валидируются до отправки на сервер
- [ ] Показывается загрузка пока данные грузятся
- [ ] Показывается ошибка если запрос упал (не молчать)
- [ ] Нет секретов в JS коде (REACT_APP_ переменные видны всем!)
- [ ] Компонент меньше 200 строк — если больше, разбить

**Чеклист React Native:**
- [ ] Работает на iOS и Android (не только на одной платформе)
- [ ] Нет жёстко заданных размеров (только flex/percentage)
- [ ] ExpoGo запускается без красного экрана

**Проверка что работает:**
- Открыть в браузере / ExpoGo
- Проверить консоль — нет ли красных ошибок
- Пройти основной сценарий руками

**Если нашёл ошибку у себя → сразу записать в `tasks/lessons.md`:**
```
### ДАТА — Что за ошибка
Что случилось: [описание]
Правило: [как не повторить]
```

## Создание React веб-приложения с нуля

Когда задача "создай веб", "инициализируй React":

```bash
npx create-react-app frontend
cd frontend
npm install react-router-dom axios zustand @tanstack/react-query react-hook-form
```

**Структура которую создаёшь:**
```
frontend/
  src/
    api/            ← функции запросов к FastAPI
      client.js     ← базовый axios instance с baseURL из .env
    components/     ← переиспользуемые компоненты
    pages/          ← страницы (по одной на роут)
    store/          ← Zustand состояние
    hooks/          ← кастомные хуки
  .env              ← REACT_APP_API_URL=http://localhost:8000
```

## Создание Expo мобильного приложения с нуля

Когда задача "создай мобилку", "инициализируй Expo":

```bash
npx create-expo-app mobile --template blank-typescript
cd mobile
npx expo install expo-router react-native-safe-area-context \
  react-native-screens expo-notifications @react-native-async-storage/async-storage
```

**Структура которую создаёшь:**
```
mobile/
  app/              ← экраны (expo-router, папки = роуты)
    (tabs)/         ← нижняя навигация
    _layout.tsx     ← корневой layout
  components/       ← переиспользуемые компоненты
  api/              ← запросы к FastAPI
    client.ts       ← fetch/axios с API_URL из Constants
  store/            ← состояние (Zustand)
  .env              ← API_URL=http://localhost:8000
```

## Примеры

**Что делаешь:**
1. WEB-SUBAGENT: компонент ProfilePage

4. Форма редактирования с валидацией
5. Описываю что получилось, как выглядит

**Что делаешь:**

2. При новом сообщении — показать уведомление
3. MOBILE-SUBAGENT: Push уведомление для мобилки
4. Показываю что уведомление приходит
