## Что анализируешь

### База данных (самое частое)
- **N+1 запросы** — цикл с запросом внутри. Fix: selectinload/joinedload в SQLAlchemy
- **Нет индекса** — EXPLAIN ANALYZE покажет Seq Scan. Fix: CREATE INDEX
- **Слишком широкий SELECT** — `SELECT *` вместо нужных колонок
- **Нет пагинации** — возвращается вся таблица. Fix: LIMIT + OFFSET или cursor
- **Размер connection pool** — слишком мало или много соединений

```sql
-- Анализ медленного запроса
EXPLAIN ANALYZE SELECT ...;

-- Медленные запросы (нужен pg_stat_statements)
SELECT query, mean_exec_time, calls
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

### FastAPI/Python
- **Sync в async** — блокирующий вызов внутри async функции. Fix: запустить в thread pool
- **Большие ответы** — сжатие gzip, убрать лишние поля из ответа
- **Нет кеша** — одни и те же данные читаются снова и снова. Fix: Redis cache
- **Много памяти** — большие объекты в памяти, утечки. Профилировать с `memory_profiler`

```python
# Проверить время выполнения
import time
start = time.time()
result = await slow_function()
print(f"Время: {time.time() - start:.3f}s")

# Запустить sync функцию в thread pool
import asyncio
result = await asyncio.get_event_loop().run_in_executor(None, sync_function)
```

### Redis
- **Cache miss rate** — кеш не работает эффективно
- **Слишком длинный TTL** — стейл данные
- **Слишком короткий TTL** — кеш бесполезен, постоянные промахи
- **Нет кеша** — добавить для часто читаемых, редко меняемых данных

### React фронтенд
- **Лишние ре-рендеры** — useMemo, useCallback, React.memo где нужно
- **Большой бандл** — code splitting, lazy loading компонентов
- **Много запросов** — объединить, добавить кеш (React Query)
- **Изображения** — сжатие, lazy loading, правильные размеры

| Уровень | Задержка | Действие |
|---------|----------|---------|
