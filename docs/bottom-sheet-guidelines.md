# Bottom Sheet Guidelines

## Основное правило

В проекте разрешён только один паттерн реализации bottom sheet.

Эталонная реализация:

Счета → Редактировать → выбор иконки

Этот bottom sheet считается source of truth для поведения.

Также корректно работают:

Настройки → Очистить аккаунт  
Редактирование операции

Они используют тот же механизм.

--------------------------------

# Запрещено

Нельзя:

- создавать новый draggable sheet внутри screen layout
- делать кастомные swipe/dismiss реализации
- рендерить sheet как часть page-screen контейнера
- позволять заднему экрану участвовать в gesture
- использовать CSS/JS костыли для исправления drag

Такие реализации приводят к:

- двойному движению экранов
- gesture leak
- сворачиванию Telegram WebView
- ошибкам Load failed

--------------------------------

# Как должен работать Bottom Sheet

Любой bottom sheet должен:

- открываться поверх экрана
- блокировать задний экран
- принимать drag только внутри sheet
- закрываться по swipe вниз
- закрываться через кнопку Close
- не резервировать место под BottomNav если он скрыт
- не двигать underlying page-screen

--------------------------------

# Как создавать новый Bottom Sheet

Если нужен новый sheet:

1. Найти существующий рабочий sheet:

Счета → выбор иконки

2. Использовать тот же:

- wrapper
- host
- gesture logic
- dismiss pipeline
- overlay
- scroll lock

3. Не создавать новый механизм.

--------------------------------

# Проверка перед коммитом

Перед завершением задачи проверить:

- двигается только sheet
- задний экран не двигается
- swipe вниз закрывает sheet
- Telegram WebView не сворачивается
- нет второго draggable слоя
- npm run build проходит

--------------------------------

# Для Codex

Перед созданием любого нового bottom sheet:

1. Найти релевантные agents и skills.
2. Найти существующую реализацию sheet выбора иконки.
3. Использовать её как базовый паттерн.
4. Не изобретать новую механику swipe/dismiss.
